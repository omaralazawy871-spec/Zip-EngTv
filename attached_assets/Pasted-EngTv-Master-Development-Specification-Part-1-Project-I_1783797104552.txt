EngTv - Master Development Specification

Part 1 - Project Identity and Mandatory Rules

IMPORTANT

You are NOT creating a new IPTV application.

You are continuing an existing project named EngTv.

The uploaded project already contains:

* Backend
* Database
* API
* Viewer application
* Admin authentication
* Routing
* Existing architecture

You MUST preserve the current architecture.

You MUST read the entire project before writing or modifying any code.

You MUST NOT replace the architecture with your own.

You MUST NOT restart the project.

You MUST NOT create another IPTV Player.

You MUST continue the existing project only.

⸻

ROLE

You are the software engineer assigned to complete EngTv.

You are NOT the product designer.

You are NOT allowed to invent new features unless explicitly requested.

You must execute the specification exactly.

⸻

PROJECT GOAL

EngTv is a professional IPTV platform with centralized management.

The viewer never manages playlists.

The administrator manages everything.

Every change performed by the administrator must automatically appear for every application user.

⸻

ABSOLUTE RULES

These rules are mandatory.

Violation of any rule is considered a project failure.

Rule 1

Do not redesign the project architecture.

Continue from the current implementation.

⸻

Rule 2

Do not convert the application into a traditional IPTV Player.

The viewer must never import:

* M3U
* Xtream
* Playlist
* URL

⸻

Rule 3

The viewer must never see IPTV settings.

No playlist management.

No source management.

No technical configuration.

⸻

Rule 4

Only one person is the Administrator.

Only the administrator has access to management functions.

⸻

Rule 5

The application must always work from the central server.

All channels come from the server.

Never from local playlists.

⸻

Rule 6

Every administrator change must instantly become available to every user after synchronization.

Examples:

Adding a channel

Deleting a channel

Editing channel

Changing logo

Changing category

Changing source

Adding new source

Disabling source

Everything must be synchronized centrally.

⸻

Rule 7

Users are viewers only.

They never configure IPTV.

⸻

Rule 8

The administrator controls:

Channels

Categories

Sources

Application settings

Branding

Synchronization

Everything.

⸻

CENTRALIZED SOURCE MANAGEMENT

The application must support multiple IPTV sources.

Supported source types:

• Xtream Codes

• M3U

Unlimited number of sources.

Each source has:

* Name
* Type
* Status
* Last synchronization
* Channel count
* Category count
* Last update
* Synchronize button
* Edit
* Delete
* Enable
* Disable

⸻

SOURCE IMPORT

When the administrator adds a source:

The system must automatically:

Download channels

Download categories

Download logos

Store everything inside the database

Make the imported channels available inside EngTv

No manual importing.

No manual channel creation.

Everything must be automatic.

⸻

MULTIPLE SOURCES

The system must support multiple active sources simultaneously.

Example:

Sports Source

Movies Source

Kids Source

News Source

Religious Source

The application merges everything into one organized platform.

The viewer never knows how many sources exist.

The viewer only sees EngTv.

⸻

DUPLICATE CHANNEL MANAGEMENT

When the same channel exists in multiple sources:

Do NOT duplicate the channel in the viewer.

The architecture should support one logical channel with one or more available stream sources if needed.

This should allow future support for backup streams instead of duplicated channels.

Do not implement shortcuts that make future expansion difficult.

Part 5 — Hidden Management System + Security + Full Source Control

Create a completely hidden internal management system that is only accessible by the developer/app owner.

The normal user must never see any indication that a management system exists.

⸻

Developer Mode

Implement a hidden Developer Mode with secure access.

Requirements:

* The developer panel must not appear anywhere in the normal user interface.
* No visible buttons, menus, or settings related to administration.
* Access should only be possible through a private method such as:
    * Secret gesture.
    * Multiple taps on the app logo.
    * Hidden developer authentication code.
    * Secure developer login.

⸻

Developer Management Dashboard

The hidden dashboard must provide complete control over application content.

⸻

IPTV Source Management

Supported sources:

M3U / M3U8

Ability to:

* Add playlist URL.
* Update playlist URL.
* Test playlist availability.
* Check channel loading status.
* Display the number of successfully loaded channels.
* Detect failed or unavailable channels.

Xtream Codes Support

* Server URL.
* Username.
* Password.

Features:

The application should:

* Test connection speed.
* Verify the source is working.
* Count available channels.
* Detect errors.

Automatic Content Updates

The application must support automatic playlist synchronization.

Features:

* Check sources automatically when the app starts.
* Update channels in the background.
* Keep the last successful channel database.
* Avoid deleting existing channels when a server temporarily fails.

Example:

If the IPTV server is offline:

* The app keeps the previous channel list.
* Shows available cached channels.
* Attempts to reconnect later automatically.
